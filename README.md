# DocViewer

Приложение для просмотра и аннотирования документов. Построено на Angular 21 с использованием standalone-компонентов и signals.

### Функцонал

- Добавление аннотации осуществляется двойным кликом/тапом.
- При фокусе на аннотации появляется функционал редактирования, перемещения, закрытия и расширения области аннотации.
- Кнопки масштабирования позволяют изменять масштаб страниц в диапазоне от 50% до 300%.

## Запуск

```bash
# Docker
docker build -t docviewer .
docker run -p 8080:80 docviewer
```

Для доступа к тестовой странице в браузере нужно ввести:
`http://localhost:8080/document/1`

или

```bash
npm run start
```

Для доступа к тестовой странице в браузере нужно ввести:
`http://localhost:4200/document/1`

## Тесты

```bash
npm run test
```

## Плюсы реализации

- **Модульная структура** - код разделён на слои `core`, `features`, `shared`, `pages` с path-алиасами, обеспечивая более короткие и простые импорты и понятную навигацию по проекту.
- **Standalone-компоненты** - отсутствие NgModules упрощает структуру и дерео зависимостей.
- **OnPush change detection** все компоненты используют `ChangeDetectionStrategy.OnPush`, что минимизирует лишние циклы проверки изменений и повышает производительность.
- **Signals для управления состоянием** - использование Angular Signals вместо подписок на Observable, простой реактивный стейт.
- **DI-токен DOCUMENT_API** - API-слой реализован через injection-токен с двумя реализациями (`DocumentHttpApi` и `DocumentMockApi`). Переключение между мок-API и реальным API происходит через переменную окружения без изменения кода компонентов.
- **Паттерн Facade** - `DocumentService` инкапсулирует работу с API, компоненты не зависят от конкретной реализации.
- **Drag-filter стратегия** - `DraggableItemDirective` принимает опциональную функцию-фильтр, позволяя кастомизировать поведение перетаскивания (например, запрет drag в области ползунка для изменения размера аннотации). Это также делает директиву независимой от конкретных компонентов.
- **Универсальная директива** - `DraggableItemDirective` не привязана к конкретному компоненту, может использоваться повторно для любых перетаскиваемых элементов.
- **Поддержка mouse и touch** - обработка `mousedown/mousemove/mouseup` и `touchstart/touchmove/touchend` позволяет работать и на десктоп-экранах, и на мобильных устройствах.
- **Процентные координаты** - позиция хранится в процентах, что обеспечивает корректное отображение при зуме.
- **Создание по двойному клику/тапу**
- **Автоподстройка высоты textarea**
- **Строгий TypeScript** `strict: true`.
- **ESLint + Prettier + Husky/Lint-staged** линтинг, форматирование и pre-commit хуки обеспечивают единообразие кода.
- **Конфигурационные константы** магические числа вынесены в отдельные файлы (`RESIZE_HANDLE_SIZE`, `DOUBLE_TAP_DELAY`).
- **Unit-тесты** охватывают все компоненты и сервисы,
- **Vitest** использование нового и более производительного Vitest.

## Минусы реализации

- Ширина страницы жёстко задана в 800px. Не особо удобно для мобильных экранов.

**Решение:** адаптировать размеры элементов под мобильные экраны, отладить поведение на тачскринах.

- `DocumentHttpApi` не реализован.

**Решение:** реализовать методы `getDocumentById()` и `saveDocument()` с использованием `HttpClient` или `resource API`, добавить `provideHttpClient()` в конфигурацию приложения.

- Отсутствуют `aria-label` у кнопки удаления и зоны ресайза. Перетаскивание невозможно с клавиатуры. Нет видимых стилей фокуса для keyboard-пользователей.

**Решение:** добавить `aria-label`, `role` атрибуты; реализовать перемещение аннотаций стрелками клавиатуры; добавить стили на фокусе.

- Нет виртуального скроллинга / ленивой загрузки страниц / API с пагинацией. Все страницы документа рендерятся одновременно. Для документов с большим количеством страниц может вызвать проблемы с производительностью.

**Решение:** использовать Virtual Scrolling и запросы с пагинацией.

- Отсутствие обратной связи при сохранении. Кнопка «Сохранить» не имеет состояний loading/disabled/success. Пользователь не понимает, произошло ли сохранение.

**Решение:** добавить состояния кнопки (`saving`, `saved`, `error`), показать snackbar/toast с результатом, либо спинер на кнопке.

- Все изменения необратимы - удалённую аннотацию или перемещение нельзя отменить.

**Решение:** добавить поддержку отмены последних действий.

## Известные проблемы

### Наложение и близость областей перетаскивания и выделения текста

Почти вся область аннотации является зоной перетаскивания, отчасти это вызывает проблему с тем, что текст внутри textarea сложно выделить мышью - drag-событие перехватывает взаимодействие раньше, либо ноаборот мешает передвигать элемент.

**Решение:** разделить режимы - перетаскивание только за заголовок или рамку аннотации, а внутренняя область отдана под редактирование текста.

### Нет ограничения длины текста аннотации

Textarea не имеет `maxlength` - пользователь может вставить неограниченно большой текст, что приведёт к разрастанию аннотации.

**Решение:** определить максимально допустимые размеры и добвить атрибут `maxlength` на textarea и ограничить максимальные размеры аннотации через CSS (`max-width`, `max-height`).

### Изменение масштаба без кнопки сброса или ручного ввода масштаба

Нет возможности быстро вернуть масштаб к исходному - при текущем шаге 0.1 нужно множество кликов для изменения масштаба осбенно при высоком увеличеннии.

**Решение:** добавить кнопку «Reset zoom». Сделать поле масштаба изменяемым в ручную. Опционально добавить сохранение пользовательской настройки масштаба.

### Нет lazy-loading

Все компоненты загружаются сразу при стате приложения. В нашем случае это некритично, но при росте приложения потребуется, чтобы уменьшить время начальной загрузки.

**Решение:** вынести маршруты в `loadComponent` / `loadChildren` для отложенной загрузки.

### Нет интернационализации (i18n)

Все строки интерфейса захардкожены на русском или английском языке.

**Решение:** подключить `@angular/localize` или `ngx-translate` и вынести строки в файлы переводов.

### Нет E2E-тестов

Отсутствуют сквозные тесты.

**Решение:** добавить Playwright или Cypress для тестирования.

### Нет тестов для drag-and-drop директивы

`DraggableItemDirective` содержит сложную логику, которую желательно конечно покрыть тестами.

**Решение:** написать unit-тесты с имитацией mouse/touch-событий, проверкой ограничения границами и корректности пересчёта позиции в проценты.

### Нет авторизации пользователя

Отсутствует авторизация пользователя.

**Решение:** добавить авторизацию пользователей.

---

Проблем достатчоно много, но поскольку ТЗ было ограниченным, то вероятно попытка решить все эти проблемы внутри данного тестового задания будет излишним. Возможно в некоторых случаях более уместен rxjs, нежели сигналы, но было желание попробовать создать приложение без использования оного.
